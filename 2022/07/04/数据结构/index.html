数据结构与算法系列

数组

数组是一种连续存储线性结构，元素类型相同，大小相等。数组尺寸不能改变，通常使用整形索引值访问元素。

优点

  • 存取速度快

缺点

  • 空间通常有限制
  • 需要大块连续的内存块
  • 插入删除元素的效率很低

Java 的 ArrayList

ArrayList 实现了 List 接口，是顺序容器，即有存放顺序的。该类与 Vector 的区别是为追求效率，未实现同步。每个 ArrayList 都有一个容量 (capacity)，表示底层数组的实际大小。当添加元素且容量不足时，容器会自动增大底层数组的大小，约为原大小的 1.5 倍。

size()，isEmpty()，get()，set() 方法均能在常数时间内完成，add() 方法的时间开销与插入位置有关，addAll() 方法的时间开销与添加元素的数量成正比。其余方法大多是线性时间。

优缺点

  • 遍历元素和随机访问元素的效率较高
  • 添加和删除需要大量移动元素，按照元素查询效率低

自动扩容

数组进行扩容时，会将老数组中的元素拷贝一份到新数组中，每次底层容量增长大约为原容量的 1.5 倍（可通过反射方式获取底层容量大小）。该操作代价很高。在实际使用时，应尽量避免容量的扩张。若可预知保存的匀速数量，在构造实例时，可指定其容量；或根据实际需求，通过调用 nesureCapacity 方法来手动增加 ArrayList 实例的容量。

add(), addAll()

在添加元素之前，都需要进行剩余空间检查。而扩容操作最终都是通过 grow() 方法完成的。

add(int index, E e) 需要先对元素进行移动，然后完成插入操作，故该方法有着线性的时间复杂度。

addAll() 方法与 add() 方法 一样，存在在末尾添加和指定位置添加的两种情况。addAll() 的时间复杂度不仅与插入元素的多少有关，也跟插入的位置有关。

set() 与 get() 不多赘述

remove()

add() 方法的逆操作过程，需要将删除点之后的元素向前移动一个位置。需要注意的是，为了让 GC 起作用，最后一个位置被显示赋值 null 值。

trimToSize()

将底层容量调整为当前列表保存的实际元素数量。

Fail-Fast 机制

ArrayList 也实现了快速失败的机制。与此对应的机制还有安全失败。

Vector 类

与 ArrayList 很相似，该类实现了一个动态数组，但两者有些不同：

  • Vector 是同步访问的
  • Vector 包含了很多传统的方法，这些方法不属于集合

自动扩容

与 ArrayList 一样，Vector 也会自动扩容，只不过每次都扩增为原容量的 2 倍。

链表

节点离散分配，彼此通过指针相连，每个节点可有一个前驱节点和后续节点。首节点没有前驱节点，尾结点没有后续节点。确定一个链表，只需要一个头指针。

优点

  • 空间没有限制
  • 内存无需连续
  • 插入删除元素的效率很高

缺点

  • 存取速度慢

Java 的 LinkedList

LinkedList 同时实现了 List 接口和 Deque 接口，即既可看作一个顺序容器，也可看作一个队列，又可看作一个栈。不过 Java 官方不再建议使用 Stack 类，且 Java 里没有 Queue 类，关于栈或队列，更建议选择 ArrayQueue。

实现方式决定了 LinkedList 所有与下标有关的操作都是线性时间，而在首段或末尾删除元素则是常数时间。为追求效率，也未实现同步。

底层数据结构

LinkedList 通过双向列表实现。双向列表的每个节点用内部类 Node 表示。LinkedList 通过 first 和 last 引用分别指向链表的第一个和最后一个元素。当链表为空时，first 和 last 都指向 null。

getFirst(), getLast()

removeFirst(), removeLast(), remove(e), remove(index)

删除元素，指的是删除第一次出现的元素。若没有该元素，则返回 false。判断的依据是 equals 方法，如果为 true，则直接 unlink 这个 node。LinkedList 也可存放及删除 null 元素。

remove(index) 使用下标计数，只需要判断该 index 是否有元素既可。

add()

add(E e) 方法在末尾插入元素，因为有 last 指向链表末尾，花费为常数时间；另一个为 add(int index, E element)，该方法在指定下标出插入元素，则需要先通过线性查找找到具体位置，再完成插入操作。

addAll() 方法并不是通过调用 add()，而是自己重新实现了一套逻辑。

clear()

为了让 GC 更快回收元素，将每个 node 之间的引用关系赋空。

查找

查找操作的本质是查找元素的下标。查找第一次出现的 index，找不到则返回 -1。

栈和队列

数组和链表是线性存储结构的基础，栈和队列是线性存储结构的应用。

栈

特点

  • 栈中数据是按照“先进后出，后进先出”方式进出栈的
  • 向栈中添加或删除数据时，只能从栈顶操作

常用三种操作：

  • push：向栈中添加元素
  • peek：返回栈顶元素
  • pop：返回并删除栈顶元素

Java 中有 Stack 类作为栈的实现。

队列

特点

  • 队列中数据时按照“先进先出，后进后出”方式进出队列的
  • 队列只允许在“队首”进行删除操作，在“队尾”进行插入操作

常用两种操作：入队和出队

Deque 与 ArrayDeque

Deque 的全称是”double ended queue”，即双端队列，既可以当栈使用，又可以当作队列使用。ArrayDeque 和 LinkedList 是 Deque 的两个通用实现。

ArrayDeque 底层通过数组实现。为了满足两端的增删需求，该数组为循环数组。ArrayDeque 为非线程安全；且该容器不允许放入 null 元素。

在该结构中，head 指向首端第一个有效元素，tail 指向尾端第一个可以插入元素的空位。因为是循环数组，head 不一定等于 0，tail 也不一定比 head 大。

addFirst()

addFirst(E e) 是在 Deque 的首端插入元素。在添加的时候，需要考虑两个问题：

  • 空间是否足够。扩容时会申请一个原数组大小 2 倍的数组；且复制时，会先将 head 右边的元素复制过去，再复制 tail 左边的元素。
  • 下标是否越界。因为代码里的操作部分包含 head- -。

addLast()

addLast(E e) 是在 Deque 的尾端插入元素。由于 tail 总是指向下一个可以插入的空位，因此只需 elements[tail] = e。插入完成后再检查空间，看情况是否进行扩容。

peekFirst() && peekLast()

返回 head 元素，或 tail 前面的 元素。

pollFirst()

删除并返回 Deque 首端元素，也就是 head 位置处的元素。

pollLast()

删除并返回 Deque 尾端元素，也就是 tail 位置前面的那个元素。

哈希

概述

根据关键码值而直接进行访问的数据结构。它通过把关键码值映射到表中一个位置来访问记录，以加快查找的速度。

哈希相比数组和链表，寻址容易，插入删除也容易。

速度快的原理

哈希在知道 key 值以后，可以直接计算出这个元素在集合中的位置，省略了繁复的查找过程。
